# 🚨 ОСНОВНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

## Проблема 1: БД не подключается на Docker-сервере ❌

### Что происходит:
- На локальном компьютере работает
- На сервере в Docker выдает ошибки подключения

### Решение КЕМ:
**Откройте `.env` файл и измените:**

```bash
# ❌ НЕПРАВИЛЬНО (для сервера):
DB_HOST=localhost

# ✅ ПРАВИЛЬНО (для Docker на сервере):
DB_HOST=db
```

### Почему:
- `localhost` = твой компьютер
- В Docker контейнер не может обращаться к твоему компьютеру как к `localhost`
- `db` = имя сервиса в `docker-compose.yml`, которое Docker автоматически разрешает

## Проблема 2: Маты не считаются 🤷

### Шаг 1: Проверьте логи на ошибки БД
```bash
docker-compose logs bot | grep "ERROR\|❌"
```

Если видите ошибки БД - смотрите **Проблема 1**.

### Шаг 2: Проверьте, обнаруживает ли бот маты
Отправьте сообщение с матом в бот и посмотрите логи:
```bash
docker-compose logs -f bot | grep "Найдены маты"
```

**Если видите такую строку:**
```
Найдены маты: [...] (всего: 3) от Владик (uid:12345)
```
→ Маты обнаруживаются. Проверьте ошибку сохранения в БД (Шаг 1).

**Если не видите эту строку:**
→ Маты в `bad_words_list.py` могут быть неправильно написаны.

### Шаг 3: Проверьте список матов
Откройте `bad_words_list.py` и убедитесь, что нужное слово там есть.

### Шаг 4: Проверьте логику поиска
Бот ищет матерные слова в тексте по двум правилам:

1. **Точное совпадение** (из `EXACT_WORDS`)
   - Слово "пиздец" = полностью совпадает → считается 1 мат
   - Слово "пиздят" = НЕ совпадает точно → не считается

2. **По корню слова** (из `ROOT_WORDS`)
   - Слово "пиздят" начинается с "пизд" → считается 1 мат ✓
   - Слово "пиздили" начинается с "пизд" → считается 1 мат ✓

**Если слово из `bad_words_list.py` не обнаруживается:**
- Может быть неправильная кодировка (ё vs e)
- Может быть слишком длинный корень

## Проблема 3: Бот вообще не запускается 💥

### Проверьте:
```bash
# Локально:
python app.py

# Docker:
docker-compose logs bot
```

### Ищите в логах эту строку:
```
✅ Бот готов к приему сообщений
```

**Если её нет:**
1. Посмотрите ошибку выше в логах
2. Кажется, это ошибка при подключении к БД или токен неправильный

### Проверьте токен:
```bash
grep BOT_TOKEN .env
```

Токен должен быть получен от `@BotFather` в Telegram.

## Проблема 4: Команда `/count` не работает ❌

### Решение:
То же самое что для Проблемы 2 - БД не подключается или нет записей.

Если БД подключена нормально, значит в базе нет записей о матах.
Сначала отправьте сообщение с матом, потом проверьте `/count`.

## 📋 Быстрая дагностика

Скопируйте-вставьте в терминал для быстрой проверки:

```bash
# 1. Проверьте подключение к БД в логах
docker-compose logs bot | head -20

# 2. Отправьте сообщение с матом в бот, потом:
docker-compose logs bot | tail -20

# 3. Проверьте, записалось ли в БД
docker-compose logs bot | grep "✓ Добавлено"

# 4. Проверьте статистику командой /count в боте
```

## 🆘 Если всё ещё не работает

1. **Перезагрузите контейнеры:**
   ```bash
   docker-compose restart
   ```

2. **Пересоздайте всё с нуля:**
   ```bash
   docker-compose down
   docker-compose up -d
   docker-compose logs -f bot
   ```

3. **Проверьте в `DEBUGGING.md` полную документацию**

4. **Включите подробное логирование SQL:**
   - Отредактируйте `database/db.py`
   - Измените `echo=False` на `echo=True`
   - Перезагрузите бота
