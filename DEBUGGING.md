# 🧹 Отладка бота для проверки матов

## 🔴 Основные проблемы и решения

### 1️⃣ БД не подключается на сервере (Docker)

**Проблема:** На локальном компьютере работает, а на сервере в Docker не работает.

**Причина:** В `.env` файле `DB_HOST=localhost`, но в Docker контейнер не может подключиться к localhost.

**Решение:**
```bash
# В .env файле измените:
# Для Docker (на сервере):
DB_HOST=db           # Имя сервиса из docker-compose.yml

# Для локального запуска:
DB_HOST=localhost    # Ваш локальный компьютер
```

**Как проверить подключение к БД:**
```bash
# Посмотрите логи бота при запуске
docker-compose logs -f bot

# Ищите строку вроде:
# 📊 Подключение к БД: postgresql+asyncpg://***:***@db:5432/mytgbot
# ✓ База данных инициализирована успешно
```

---

### 2️⃣ Маты не считаются

**Проблема:** Бот получает сообщения, но не считает маты.

**Возможные причины и проверка:**

#### A) Проверьте логи на предмет ошибок БД:
```bash
# Локальный запуск
python app.py

# Docker
docker-compose logs -f bot | grep "❌\|✗\|Error"
```

#### B) Проверьте, обнаруживает ли бот маты:
Ищите в логах строки вроде:
```
Найдены маты: [...] (всего: N) от ИмяПользователя (uid:...)
✓ Добавлено N матов в БД от ИмяПользователя
```

Если нет этих строк - **маты не обнаруживаются**.

#### C) Проверьте список матов:
```bash
# Посчитайте слова в bad_words_list.py
grep -c '\"' bad_words_list.py
```

Если список содержит нужный мат - проверьте регулярное выражение.

#### D) Проверьте регулярное выражение:
В `handlers/user_handler.py` используется паттерн `r"[а-яёa-z0-9]+"`
- ✓ Правильно разбивает текст на слова
- ⚠️ Может не разбивать слова с дефисом (хуй-хуй) на отдельные слова

**Тест в Python:**
```python
import re
text = "ебать это дерьмо"
words = re.findall(r"[а-яёa-z0-9]+", text.lower())
print(words)  # ['ебать', 'это', 'дерьмо']
```

### 3️⃣ Бот получает сообщения, но не реагирует

**Причина:** Обычно проблема с подключением к БД.

**Проверка:**
```bash
# В логах должны быть строки вроде:
# ✓ Планировщик запущен
# ✅ Бот готов к приему сообщений
```

Если этих строк нет - значит бот не запустился.

## 📊 Как включить детальное логирование

Если информация в логах недостаточна, отредактируйте `database/db.py`:

```python
# Вместо этого:
engine = create_async_engine(DATABASE_URL, echo=False)

# Напишите это:
engine = create_async_engine(DATABASE_URL, echo=True)
```

Это выведет все SQL запросы в консоль (очень подробно).

## 🔧 Docker-compose команды для отладки

```bash
# Запустить бота
docker-compose up -d

# Посмотреть логи бота в реальном времени
docker-compose logs -f bot

# Посмотреть только ошибки
docker-compose logs bot | grep "ERROR\|❌\|✗"

# Войти в контейнер бота
docker-compose exec bot bash

# Перезагрузить контейнер
docker-compose restart bot

# Полностью удалить контейнеры и переоздать
docker-compose down
docker-compose up -d
```

## 📝 Структура логов

Каждый лог содержит:
- **Время**: `2024-03-23 15:30:45,123`
- **Модуль**: `app`, `handlers.user_handler`, `database.orm_query`
- **Уровень**: `INFO` ✓, `ERROR` ❌, `DEBUG` 🔍
- **Сообщение**: Описание события

Пример:
```
2024-03-23 15:30:45,123 - app - INFO - 🚀 Инициализация базы данных...
2024-03-23 15:30:45,456 - database.db - INFO - 📊 Подключение к БД: postgresql+asyncpg://***:***@localhost:5432/mytgbot
2024-03-23 15:30:46,123 - app - INFO - ✓ База данных инициализирована успешно
```
